The macro RECOVER

The MATLAB macro RECOVER can be called if, after running a simulation, the results have not properly been sent to the MATLAB workspace. This sometimes happens due to a bug in SIMULINK, which apparantly prevents results to be transferred to the Workspace when To Workspace blocks are used in deeper levels of graphical systems (the bug has been encountered in several SIMULINK versions up to 1.2c). RECOVER basically makes one more call to the S-function in order to re-initialize the system. This often helps to restore the 'lost' results, without repeating the simulation itself.

Type recover('sfunc') at the command-line if you want to rescue simulation results for the S-function sfunc.
