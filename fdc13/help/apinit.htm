The autopilot initialization routine APINIT

The MATLAB program APINIT can be used to initialize the autopilot systems APILOT1, APILOT2, APILOT3, and the non-linear aircraft model Beaver. This routine needs to be runned before starting a new simulation and before editing the system (the latter is not really necessary, although SIMULINK will sometimes complain about missing parameters if the system is edited without having initialized the system).

The program can be started by typing apinit at the MATLAB command-line, or by double-clicking the APINIT button in the autopilot systems. First, a graphical user-menu will be displayed. The user must click the buttons in the appropriate order and enter all necessary values at the MATLAB command-line (for the simplified system APILOT1 the VOR/ILS initialization may be skipped). APINIT calls several subroutines; see the source-code of APINIT.M in the FDC directory APILOT for more details.

Note: APINIT does not initialize the autopilot mode and reference values of the control inputs. Those values should be defined with the routine APMODE.
