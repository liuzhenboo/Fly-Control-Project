The blocklibrary NAVLIB

The FDC blocklibrary NAVLIB contains radio-navigation blocks for SIMULINK. Currently, there are two sublibraries, one for ILS signals and one for VOR signals. ILS stands for Instrument Landing System; VOR means Very high frequency Omnidirectional Range. This library may be enhanced with other radio-navigation blocks, such as Global Positioning System. A detailed theoretical description about these simulation models can be found in the references at the end of this help-text.

The ILS sublibrary contains the following blocks:
  • ILS (Nominal ILS signals)
         ILS signals without noise and/or steady-state errors
  • GSerr
         Glideslope steady-state errors according to ICAO guidelines
  • GSnoise1
         Glideslope noise according to AGARD R-632 (ref.[3])
  • GSnoise2
         Glideslope noise according to NASA CR-2022 (ref.[4])
  • LOCerr
         Localizer steady-state errors according to ICAO guidelines
  • LOCnoise1
         Localizer noise according to AGARD R-632 (ref.[3])
  • LOCnoise2
         Localizer noise according to NASA CR-2022 (ref.[4])
  • ILS example
         Example system which demonstrates the practical use of the other ILS blocks

The VOR sublibrary contains the following blocks:
  • VOR (Nominal VOR signals)
         VOR signals without error components
  • VORerr
         Steady-state error for the VOR signal
  • VOR example
         Example system which demonstrates the practical use of the other VOR blocks

Note: currently there are no accurate noise models available for the VOR blocks. Should such models become available, they can easily be included in the VOR sublibrary of NAVLIB.
